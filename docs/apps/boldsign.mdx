> Boldsign uses OAUTH2 for authentication.
## 📔 Documentation

- [OAuth Documentation](https://developers.boldsign.com/authentication/OAuth-2.0/?region=us)
- [API Documentation](https://developers.boldsign.com/api-overview/getting-started/?region=us)

## ⚙️ Configuration

<Tip>New app can be created on [Boldsign OAuth Apps](https://app.boldsign.com/api-management/oauth-apps/).</Tip>

Once you create the App, please configure the below Redirect URL:


```
https://backend.composio.dev/api/v1/auth-apps/add
```

After registering the app, copy the Client ID from the app page and generate/copy the Client Secret. On Composio's [Integrations](https://app.composio.dev/apps) page, find Boldsign and add a new integration. Paste your Client ID and Secret, then Save to connect Boldsign.


## ⚒️ Actions

Boldsign actions are not supported yet. In case, you’re looking for Boldsign actions, please write to us at tech@composio.dev.

## 🪝 Triggers

Boldsign triggers are not supported yet. In case, you’re looking for Boldsign Triggers, please write to us at tech@composio.dev.
