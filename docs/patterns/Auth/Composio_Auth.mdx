Composio allows you to create a fully customized authentication experience for your users while leveraging our powerful backend infrastructure.

## Implementing Custom Authentication

### Using Your Own Developer App

1. Create your developer app in the third-party service (e.g., GitHub, Google).
2. Configure the redirect URL in your developer app settings to match Composio's requirements.
3. Use the obtained client ID and secret when setting up your Composio integration.

### Specifying a Custom Redirect URL

When initiating a new connection, you can specify a custom redirect URL to ensure users are directed back to the appropriate page in your application after the authentication flow completes.

<CodeGroup>

```python Python
from composio import ComposioToolSet

composio = ComposioToolSet(api_key="YOUR_API_KEY")

request = composio.connectedAccounts.initiate(
    integrationId="YOUR_INTEGRATION_ID",
    redirectUrl="https://your-app.com/auth/callback",
    entityId="user@example.com",
    data={
        "subdomain": "your_subdomain"
    }
)

print("Redirect URL:", request.redirectUrl)
```

```javascript JavaScript
import { OpenAIToolSet } from "composio-core";

const composio = new OpenAIToolSet({ apiKey: "YOUR_API_KEY" });

const connectedAccountRequest = await composio.connectedAccounts.initiate({
    integrationId: "YOUR_INTEGRATION_ID",
    redirectUrl: "https://your-app.com/auth/callback",
    entityId: "user@example.com",
    data: {
        subdomain: "your_subdomain"
    }
});

console.log("Redirect URL:", connectedAccountRequest.redirectUrl);
```

</CodeGroup>

### Handling the Redirect

After the authentication process, users are redirected to your specified URL with the following query parameters:

| Parameter | Description |
|-----------|-------------|
| `status` | Either 'ACTIVE' or 'ERROR' |
| `connectedAccountId` | The unique ID for this user's connected account in Composio |
| `appName` | The name of the connected application |

Use these parameters to update your application's state and provide appropriate feedback to the user.
