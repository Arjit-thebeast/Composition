With Composio you can build your own Actions under existing tools, such actions are called Custom Actions.
You can build these Actions without having to worry about Authentication.

In this example, you can see how to create a Custom Action with Gmail.

<Steps>

<Step title="Import required dependencies">
```python Python
from composio import ComposioToolSet, App, action
import requests
import datetime

toolset = ComposioToolSet()
```
</Step>

<Step title="Creating our Custom Action">
```python Python
@action(toolname="gmail")
def get_emails(auth: dict, query:str) -> str:
    """
    Get gmail account information.
    :param auth: Authentication credentials
    :param query: Gmail query (provide a description here) 
    :return info: Gmail account information
    """
    current_dateTime = datetime.datetime.now().isoformat() + 'Z'  
    tomorrow_dateTime = (datetime.datetime.now() + datetime.timedelta(days=1)).isoformat() + 'Z' 
    base_url = auth['base_url']
    url = f"{base_url}/gmail/v1/users/me/messages?q=in:sent after:{current_dateTime} before:{tomorrow_dateTime}"

    headers = auth['headers']
    query_params = auth['query_params']
    response = requests.get(url, headers=headers)

    if response.status_code == 200:
        print("Messages:", response.json())
    else:
        print("Error:", response.status_code, response.json())
    return str(response)
```
</Step>

<Step title="Execute the Action">
```python Python
print(
    toolset.execute_action(
        action=get_emails,
        params={
            "query":"Newsletters" #this is how you can pass parameters, 
            # add :params <arg_name>: <description> in the custom action doc string
        },
    )
)

```
</Step>
</Steps>

**Congratulations!** You just created and executed a Custom Action under Gmail!

## Advantages of Custom Actions

- **Flexibility**: Custom Actions allow you to tailor functionalities to meet specific needs, enabling unique workflows that standard actions may not cover.

- **Ease of Use**: You can create and manage actions without needing to handle complex authentication processes, simplifying the development process.

- **Integration**: Custom Actions can seamlessly integrate with existing tools, enhancing their capabilities and allowing for more efficient task execution.
