## Using Actions

After successfully connecting an account to Composio through integration, a unique `connectedAccountId` is generated. We will execute actions using this ID.

## **Understanding Connected Accounts**

Connected Accounts are established links between Composio and an external application, created during the user authentication process. These accounts store essential authentication details such as access tokens and refresh tokens.

## **Types of Actions**

1. **Default Actions**

   Composio supports a variety of default actions for each supported application.

2. **Custom Actions**

   Our platform offers an extensible interface for defining custom actions for both custom and default applications. [Learn more.](http://Google.com)

3. **Local Code Execution**

   Composio enables fetching authentication parameters for any connection, allowing you to execute code remotely and interact with API endpoints.

   Example: Use the `getConnection` endpoint to call GitHub, create an issue on your server, and fetch the access token. [Learn more](http://google.com)

### **Retrieving Actions**

Call `/actions` to get list of all Actions on Composio Platform. (Default + Custom)

```python
from composio.sdk import ComposioSdk

composio = Composio("api-key")

print(composio.get_list_of_actions())
```

The response will include the action's input/output schema in JSON compatible manner. It will also contain metadata like action name, ID.

After a user connects an account using an integration, a `connectedAccountId` is obtained. An action can be executed on the app for which connected account was created.

### **Step 2: We need Input Parameters to execute action**

```python
connected_account = composio.get_connected_account(connection_id="9c8e6e42-3837-4014-bd80-12696bd758f1")
params = {"repo":"speedy","owner":"utkarsh-dixit","title":"Too much light","body":"Show my dark night"}
execution_details = connected_account.execute_action(action_name=Action.GITHUB_CREATE_ISSUE, input_data=params)
```

When executed successfully, action's response will be output. Output schema as returned by the /actions endpoint.

### Retrieving All Supported Actions with Composio

```pythone
composio.getAllTools(uuid) # pick all Tools that are relevant for user.
connected_account.get_all_actions() # Get all tools from connection
```

This approach ensures that users can easily discover and utilize the full range of actions available to them, enhancing their integration and automation capabilities.
