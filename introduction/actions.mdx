# Using Actions

User was able to connect his account to Composio using integration and `connectedAccountId` was generated. Letâ€™s use it to now execute an action.

### **Accounts**

Connected Accounts represent links created when an Integration is used to authenticate a user. It stores the access token, refresh token and other parameters necessary to authenticate the user.

### **Available Action Types**

1. **Default Actions**

   For each Default App, we support many default actions.

2. **Custom Actions**

   We provide an extensible interface which allows you to define a custom action either for a custom or default app through code. [Read here.](http://Google.com)

3. **Local Code - Action**

   We also provide you an interface to fetch the auth parameters for any connection and execute the code remotely to call the API endpoints.

   Ex - Call Github Create a new issue on your server and fetch access token by using `getConnection` endpoint. [Read here](http://google.com)

### **List Default Actions**

Call `/actions` to get list of all Actions on Composio Platform. (Default + Custom)

```python
from composio.sdk import ComposioSdk

composio = Composio("api-key")

print(composio.get_list_of_actions())
```

- The response will include the action's input/output schema in JSON compatible manner. It will also contain metadata like action name, ID.

from composio.sdk.enums import Action

composio.get_list_of_actions(action_names = [Action.GITHUB_CREATE_ISSUE])

After a user connects an account using an integration, a `connectedAccountId` is obtained. An action can be executed on the app for which connected account was created.

### **Step 2: We need Input Parameters to execute action**

```python
account = composio.get_connected_account(connection_id="9c8e6e42-3837-4014-bd80-12696bd758f1")
params = {"repo":"speedy","owner":"utkarsh-dixit","title":"Too much light","body":"Show my dark night"}
execution_details = account.execute_action(action_name=Action.GITHUB_CREATE_ISSUE, input_data=params)
```

When executed successfully, action's response will be output. Output schema as returned by the /actions endpoint.
