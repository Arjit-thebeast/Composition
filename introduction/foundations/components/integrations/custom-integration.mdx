Creating custom tools on Composio is straightforward. This guide will walk you through the process of building GitHub as a custom integration. Although GitHub is already offered as a tool on Composio, this example will help you understand how to create your own integrations.

For enterprise users, we recommend reaching out to us at [tech@composio.dev](mailto:tech@composio.dev) for custom solutions.

<Steps>

<Step title="Create a Github Repository">
    The repo can be public or private. This repo will house all the files for your custom tools. You can also fork our <a href="https://github.com/SamparkAI/test-custom-repo">starter repo</a> for custom tools.
</Step>


<Step title="Create a Folder in the Repository">
    Inside the repo, you will need to create a dedicated folder for each of your custom tools. In this case, to create a GitHub integration, you will create a folder named `github`.
</Step>


<Step title="Create the integrations.yaml File">

Let's start with creating the yaml file for GitHub by adding the basic info.
name: GitHub
description: Integrate GitHub to manage your repositories directly from Composio.
logo: https://github.githubassets.com/assets/GitHub-Mark-ea2971cee799.png
categories: ["developer tools", "version control"]

auth_schemes:
```

Now you will need to determine the supported auth method by the custom tool that you're trying to add. In the most of the cases, you can figure this out with a sinmple google search "custom_tool_name auth docs", ideally first url that comes up will be the auth docs for the tool.

In this case,GitHub uses OAuth2 for authentication. 

```yaml
name: GitHub
description: Integrate GitHub to manage your repositories directly from Composio.
logo: https://github.githubassets.com/assets/GitHub-Mark-ea2971cee799.png
categories: ["developer tools", "version control"]

auth_schemes:
  - scheme_name: github_oauth2
    auth_mode: OAUTH2
    authorization_url: "https://github.com/login/oauth/authorize"
    token_url: "https://github.com/login/oauth/access_token"
    default_scopes: ["repo", "admin:org"]
    token_response_metadata: ["access_token", "token_type", "scope"]
    token_params:
      grant_type: authorization_code
    authorization_params:
      response_type: code
    proxy:
      base_url: "{{base_url}}"
      state: "{{state}}"
    fields:
      - name: base_url
        displayName: Base URL
        description: "The API base URL for GitHub"
        type: string
        default: https://api.github.com
```

Now that you have the `integrations.yaml` file ready, push the code and copy the URL of the repository.

</Step>


<Step title="Add your Custom Tool on Composio">

Go to the <a href="https://app.composio.dev/api_keys">settings</a> page on Composio and paste your repository URL. 
<Frame>
  <img src="/media/custom.gif" />
</Frame>
If your repo is private, your URL should include your GitHub personal token: `https://ghp_xxxxxxx@github.com/username/repo_name`.
Click on the "update and sync" button to test.

</Step>


<Step title="Testing Your Custom tool on Composio">

Navigate back to the tools catalogue page on Composio. Search for the GitHub tool you just created, click on it, and proceed to connect your account.

</Step>

</Steps>
